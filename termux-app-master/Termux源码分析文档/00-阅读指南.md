
## 🔑 核心要点

### 三个关键认知

1. **Termux 不是虚拟化**
   - Android 本身就是 Linux
   - Termux 只是提供了 Linux 用户空间工具
   - 程序原生运行，无性能损失

2. **Bootstrap 是核心**
   - 包含 bash, apt, coreutils 等基础工具
   - 编译时嵌入 APK
   - 首次启动解压到应用目录

3. **PTY 是关键**
   - 让程序以为在真实终端中运行
   - 支持所有终端特性
   - 通过 JNI 调用 Linux 系统调用

### 技术栈

```
应用层：Java/Kotlin (Android)
  ↓
JNI 层：C (PTY 管理)
  ↓
内核层：Linux Kernel (Android 提供)
  ↓
用户空间：Linux Tools (Bootstrap 提供)
```

## 🎯 常见问题

**Q: Termux 需要 root 吗？**
A: 不需要。Termux 运行在应用沙箱内，不需要 root 权限。

**Q: Termux 是虚拟机吗？**
A: 不是。Termux 直接在 Android 的 Linux 内核上运行，没有虚拟化。

**Q: 为什么 APK 这么大？**
A: 因为 Bootstrap 文件（约 30MB）被嵌入在 APK 中。

**Q: 可以运行所有 Linux 程序吗？**
A: 不能。只能运行为 Termux 编译的程序，且受 Android 沙箱限制。

**Q: 如何学习源码？**
A: 按照 05-源码路径导航.md 的建议路径，从简单的类开始。
