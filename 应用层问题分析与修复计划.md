## P0 — 安全与数据一致性

### 问题 1：双重配置仓库

**现象**:
- `AppConfig` 使用 `EncryptedConfigRepository`（加密 SharedPreferences）
- `ProviderManager` 使用 `SharedPreferencesConfigRepository`（明文 SharedPreferences）
- 二者读写不同的 SharedPreferences 文件，同一配置项数据不同步

**影响范围**: 17+ 个 Activity 通过 ProviderManager 读写，ChatViewModel 通过 AppConfig 读写

**修复方案**: ProviderManager 改为委托 AppConfig.configRepo()，废弃独立的 SharedPreferencesConfigRepository 实例

### 问题 2：加密回退明文

**现象**:
```kotlin
private val encryptedPrefs: SharedPreferences = try {
    EncryptedSharedPreferences.create(...)
} catch (_: Exception) {
    ctx.getSharedPreferences("jasmine_encrypted_prefs_fallback", Context.MODE_PRIVATE)
}
```
加密失败时静默回退到明文 SharedPreferences，用户完全不知道密钥暴露。

**修复方案**: 回退时记录错误日志并通过 Toast/Dialog 通知用户

### 问题 3：ProviderManager 明文存储

**现象**: ProviderManager 使用 SharedPreferencesConfigRepository，API Key 等敏感数据以明文存储

**修复方案**: 随问题 1 一并修复（统一使用加密仓库）

## P1 — 架构问题

### 问题 4：chatItems 与 ChatUiState.messages 双轨状态

**现象**:
- `ChatViewModel.chatItems` 是 `mutableStateListOf<ChatItem>()`，由 ChatStateManager 直接修改
- `ChatUiState.messages` 是 `List<ChatItem>`，从未被同步
- UI 实际使用 `viewModel.chatItems`，`uiState.messages` 是死字段

**修复方案**: 移除 ChatUiState.messages 死字段，明确 chatItems 为唯一数据源

### 问题 5：ViewModel 持有 Activity 引用

**现象**:
- `ChatViewModel.activityContext` 持有 Activity 引用
- `DialogHandlers.register(activity, builder)` 将 Activity 传入 lambda 闭包
- 闭包被 ToolRegistryBuilder 持有，生命周期长于 Activity

**修复方案**: DialogHandlers 的 AlertDialog 迁移为 Compose Dialog 状态驱动模式

### 问题 6：无 DI 框架

**现象**: 全靠手工单例 + lateinit，ViewModel 无法单元测试

**修复方案**: 后续评估引入 Koin/Hilt

## P2 — 用户体验与架构一致性

### 问题 7：32 个独立 Activity

**现象**: 每个设置子页面都是独立 Activity，跳转开销大，无法共享状态

**修复方案**: Single-Activity + Navigation Compose（已实施）

### 问题 8：MainActivity 混合布局

**现象**: MainActivity 用 AppCompatActivity + XML DrawerLayout 嵌套 ComposeView

**修复方案**: 已迁移为 ComponentActivity + setContent，双侧 Compose 抽屉布局（FileTreeComposable + 设置入口），删除 activity_main.xml、item_file_tree.xml、FileTreeAdapter.kt

---

## P3 — 代码质量

### 问题 9：包结构扁平

**现象**: 根包下 32+ 文件，Activity、单例、工具类混杂

### 问题 10：Markdown AndroidView 渲染

**现象**: 每个 AI 消息气泡 inflate 一个 TextView，长对话中性能不佳
